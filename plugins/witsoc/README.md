# Witsoc — Semiformal Math

A semiformal proof system. Author `.wit` proofs with structured steps,
check their structure, generate LLM-ready verification contexts, persist
verdict receipts. Ships a working-memory queue (`.soc` files) for
incremental research.

## When to use

- The user wants a proof of a math claim.
- The user has a `.wit` file and wants it verified.
- You want to track ongoing math/research as a queue of verifiable claims.

## CLI

All subcommands go through the dispatcher:

```bash
"$PLANE_TOOL_BIN" plugins bash witsoc <subcmd> [args...]
```

| Subcmd | Use |
| --- | --- |
| `info` | One-line status (venv installed?, wit version). |
| `list [dir]` | List `.wit` and `.soc` files under `dir` (default: worktree). |
| `new <name> [dir]` | Scaffold a fresh `.wit` skeleton (Status, MODULE, THEOREM, PROOF). |
| `check <file.wit>` | Structural check — labels, citations, missing steps. Fast, no LLM. |
| `verify <file.wit> [--step N.M]` | Emit LLM-ready verification contexts per step. Pipe the LLM's verdicts back through `receipt`. |
| `context <file.wit>` | Emit the dependency contexts without the verdict-paste markers. |
| `receipt <file.wit>` | Read verifier output from stdin, persist verdicts to `<file>.wit.receipt.json`. |
| `soc-init [file.soc]` | Scaffold a `research.soc` working-memory file. |
| `soc-show <file.soc>` | Print the structured working memory. |

## Verification workflow

1. `wit-new theorem_name` → creates `theorem_name.wit` skeleton.
2. Author the proof. Each step is `[N] HAVE/SHOW/... BY [citations]`.
3. `wit-check theorem_name.wit` → confirms structure is valid.
4. `wit-verify theorem_name.wit` → prints per-step contexts ready for LLM review.
5. Paste contexts to your reviewer (the chat above), get back verdicts.
6. Pipe verdicts into `wit-receipt theorem_name.wit` → persists `.wit.receipt.json`.

The iframe (`plugins iframe use witsoc`) shows two tabs:

- **Proofs** — file list, syntax-highlighted source, per-step tree with verdict dots, verdict-paste textarea.
- **Soc Memory** — Goal / Progress / Queue / Insights / Log cards for the `.soc` working memory.

## Iframe commands

Push from the agent:

```bash
"$PLANE_TOOL_BIN" plugins iframe-bash witsoc --command open    --args '["theorem.wit"]'
"$PLANE_TOOL_BIN" plugins iframe-bash witsoc --command focus   --args '["3.1"]'
"$PLANE_TOOL_BIN" plugins iframe-bash witsoc --command check
"$PLANE_TOOL_BIN" plugins iframe-bash witsoc --command verify
"$PLANE_TOOL_BIN" plugins iframe-bash witsoc --command tab     --args '["soc"]'
```

## First-run setup

`bin/ensure-deps` creates `data/venv/` and installs the `wit-lang` Python
package. Source resolution: `$WITSOC_SOURCE`, then `~/Development/witsoc/witsoc/`,
then PyPI (typically not yet available). Setup is one-time, ~20 seconds.

## File formats

- `.wit` — proof source. Markdown-like with required preamble (`-- Status:`,
  `MODULE`, `THEOREM`) and labeled steps (`[1]`, `[2.1]`, etc.).
- `.soc` — working memory. Structured Markdown with sections `GOAL`,
  `PROGRESS`, `INSIGHTS_SINCE_LAST_UPDATE`, `QUEUE`, `INSIGHTS`, `LOG`.
- `.wit.receipt.json` — persisted verdicts. Generated by `wit-receipt`,
  never authored by hand.
