# Literature Triage

Use this for serious open-problem work before theorem search.

## Source Stack

Prefer sources in this order:

1. original paper/problem list,
2. author-maintained page,
3. current survey,
4. recent preprint with clear theorem statement,
5. formal library,
6. database/pointer,
7. blog/forum/informal note.

## Triage Record

```markdown
### Literature Triage
- Problem origin:
- Exact statement variants:
- Best known positive results:
- Best known negative examples:
- Known barriers:
- Known failed methods:
- Current status:
- Sources checked:
- Unchecked leads:
```

## Rules

- Never transfer a solved neighboring variant to the target without variant alignment.
- Track dates for status claims.
- Record "unknown after search" separately from "open".
- If a source gives a theorem, route it through `theorem_retrieval_engine.md`.
- If literature exposes a known barrier, update `barriers.md` and `.soc`.
